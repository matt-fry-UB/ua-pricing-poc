URBAN AIR Simplified Pricing QA Tools

Instructions

Use this page to sync the data on UA Simplified Pricing QA Checklist. Uncheck the "Write to Smartsheet" box to do a dry run and preview the changes, but there's no real downside to just letting it write.

Sync Smartsheet

If any changes are made to the Simplified Pricing Rollout (2026) sheet, then run the Sync Smartsheet process. For the most part, that source sheet should not change, so you probably won't need to run this.

Run QA Checklist

Once you hear from the CC team that a location is ready for web QA, you can run the QA Checklist process. Check the box for any location(s) in the list and click the Run Pricing QA button. Then wait for the process to finish — it should take 3–5 minutes depending on how many locations you check.

Once it is complete, open the UA Simplified Pricing QA Checklist and look for any rows with mismatches indicated. If there is no mismatch for an item, you will also need to manually check the box in the "Done" column to mark it as complete.

For any items with a mismatch, use the link in the "Edit Location" column to open the section inside WordPress where you can search for and edit that location.

After you update a location (or set of locations) in WordPress, you will need to clear the caches using the "WP Rocker", "Clear Divi Cache", and "WP Engine Quick Links" tabs at the top of the page. Wait 2–5 minutes for the caches to fully clear.

After you clear the caches, you will need to refresh the Next.js pages using the Refresh the Urban Air website button. This process will also take 2–5 minutes.

Once the pages have refreshed and your WordPress updates are visible on the live site, run the QA Checklist process again to make sure the mismatched items have been corrected.

Repeat this process until all locations are green.

Sync Smartsheet

Pulls the latest reference values from the Simplified Pricing Rollout sheet and Birthday Grid, then updates the QA Checklist rows. Run this when source data changes (new locations, price corrections). Does not touch the Command Center or Website columns.
